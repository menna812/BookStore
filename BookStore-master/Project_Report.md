# BookStore Management System – Project Report

## 1. Implemented Features

- **Book Inventory Management:**
  - Add, update, and delete books.
  - Manage stock, price, and publisher details.
- **Automated Publisher Orders:**
  - When a book’s stock drops below its threshold, a database trigger automatically creates a pending order to the publisher (no duplicates for the same book/publisher while pending).
- **Customer Management:**
  - Add, update, and delete customer records.
- **Order Processing:**
  - Place customer orders, update stock, and track order status.
- **Sales & Analytics Reports:**
  - Last month’s total sales.
  - Sales on a specific day.
  - Top 5 customers in the last 3 months.
  - Top 10 selling books in the last 3 months.
  - Book replenishment count (number of times a book was reordered from the publisher).
- **User-Friendly Error Handling:**
  - All API endpoints return clear, descriptive error messages for invalid input or missing data.
- **Frontend (React):**
  - User interfaces for managing books, customers, orders, and viewing reports, with clear navigation and feedback.

## 2. ERD Diagram

```
[BOOK]---(ISBN)---<ORDER_ITEM>---(order_id)---[ORDER]---(customer_id)---[CUSTOMER]
   |                        |                          |
   |                        |                          |
(Publisher_id)         (ISBN)                     (admin_id)
   |                        |                          |
[PUBLISHER]           [ORDER_PUB_ITEM]---(order_pub_id)---[ORDER_PUB]
```

- **BOOK**: ISBN (PK), Title, Author, Publisher_id (FK), Price, Stock, Threshold
- **ORDER**: order_id (PK), customer_id (FK), admin_id (FK), order_date, total_amount
- **ORDER_ITEM**: order_item_id (PK), order_id (FK), ISBN (FK), quantity, price
- **CUSTOMER**: customer_id (PK), firstname, lastname, email, phone
- **PUBLISHER**: publisher_id (PK), name, contact_info
- **ORDER_PUB**: order_pub_id (PK), admin_id (FK), publisher_id (FK), status, constant_quantity
- **ORDER_PUB_ITEM**: order_pub_item_id (PK), order_pub_id (FK), ISBN (FK), quantity

## 3. Relational Schema

- **BOOK**(`ISBN`, Title, Author, Publisher_id, Price, Stock_Quantity, Threshold)
- **PUBLISHER**(`Publisher_id`, Name, Contact_Info)
- **CUSTOMER**(`Customer_id`, Firstname, Lastname, Email, Phone)
- **ORDER**(`Order_id`, Customer_id, Admin_id, Order_Date, Total_Amount)
- **ORDER_ITEM**(`Order_Item_id`, Order_id, ISBN, Quantity, Price)
- **ORDER_PUB**(`Order_Pub_id`, Admin_id, Publisher_id, Status, Constant_Quantity)
- **ORDER_PUB_ITEM**(`Order_Pub_Item_id`, Order_Pub_id, ISBN, Quantity)

## 4. User Interface Logic Description

- **Login Screen:** Authenticates admin users to access the system.
- **Dashboard:** Displays quick stats (total books, sales, pending orders) and navigation to all modules.
- **Books Management:**
  - List all books with search/filter.
  - Add/Edit/Delete book details.
  - Shows current stock and threshold.
  - Triggers automated publisher order if stock falls below threshold.
- **Customers Management:**
  - List, add, edit, and delete customers.
  - View customer order history.
- **Orders Management:**
  - Create new customer orders.
  - Add books to order, set quantities.
  - Updates book stock upon order completion.
  - View and update order status.
- **Publisher Orders:**
  - View all publisher orders (auto-generated by trigger).
  - Mark orders as received to update stock.
- **Reports:**
  - Select and view sales reports (last month, by day).
  - View top customers and top selling books.
  - Enter ISBN to view replenishment count for a book.
  - All reports handle errors gracefully and display clear messages if data is missing or invalid.
